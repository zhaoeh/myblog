# 1. java中的null
## 1.1 java中的null本质是什么？
1. null是java中的关键字，像public,static,final等一样，因此普通变量不能使用"null"作为标识符名称。  
2. null是大小写敏感的，因此不能将null写成NULL或者Null等，如果写错编译器将无法识别而报错。  
3. null不是一个对象，也不是一种数据类型。null本身是一个值。就像基本类型存在默认值一样，比如int默认值是0，boolean默认值是false，null是任何引用类型的默认值。即java中将null作为任何引用变量的默认值。  
4. 再次强调，null既不是对象也不是一种数据类型，它仅仅是一种特殊的值，可以将null值赋值给任何引用类型，也可以将null转换为任何引用类型。  
    比如：  
    ``` java
    String str = null;
    Integer in = null;  
    Double do = null;  
    // 将null强制转换为String类型  
    String myStr = (String)null; 
    // 将null强制转换为Integer类型  
    Integer myIn = (Integer)null; 
    ```  
    将null强制转换为任何引用类型都是可行的，编译器不会报错，并且不会抛出空指针异常。  

## 1.2 深入null值
- null值可以赋值给任何引用变量，但不能直接将null值赋值给基本类型，比如 int i = null;如果这样做了，编译器将报错。  
   但是可以将null值先赋值给对应的包装类型，比如 Integer i = null;再将包装类型赋值给基本类型，即 int a = i;实际上在这一步会发生自动拆箱操作，JVM会自动对包装类型进行拆箱然后赋值给对应的基本类型。  
   尽管可以这样做，然而在运行期间将会得到一个空指针异常。  
- 如上所述，任何含有null值的包装类，在通过java自动拆箱操作时，都会抛出一个空指针异常（NPE）。  
   有很多人认为，java在自动拆箱时，会自动将null值拆箱成对应基本类型的默认值，比如 int 类型的话会将null值拆箱为0，boolean 类型的话会拆箱为false。实际上并不会。  
   特别需要注意：Integer a = null; int b = a + 1;像这种，对null值的包装对象和基本类型进行加减法，实际上首先会对Integer进行拆箱，一拆箱就报空指针异常了。  
- null值 instanceof任何数据类型，都会返回false。即null值不是一种类型。  
   同理，将null值赋值给某个引用类型，然后instanceof该类型，也返回false。  
   如下：  
   ```java
   Integer nullInt = null;
   // 将返回false。
   nullInt instanceof Integer 
   ```  
- 众所周知，当外部通过某个类的实例对象调用某个类的实例方法（非静态方法）时，如果该实例对象为null值，将抛出空指针异常，比如存在Class Test：  
    ```java
    Class Test{
        public void eat(){
            System.out.println("eat");
        }

        public static void drink(){
            System.out.println("drink");
        }
    }
    Test test = null;
    // 调用test对象的实例方法eat()，将抛出空指针异常。
    test.eat(); 
    ```
    但我们绝对不会想到，当调用一个null值的引用对象的静态方法时，并不会抛出空指针异常。如下：  
    ```java
    Test test = null;
    // 调用test对象的类方法drink()，调用成功。
    test.drink(); 
    ```
    这是因为，静态成员和静态方法是属于当前类的，初始化阶段早，并且存于JVM方法区内存的静态区，属于共享数据，采用静态绑定，因此并不会抛出空指针异常。  
- null值可以传递给任何通过引用类型参数进行接收的方法，比如 public void print(Object obj);从编译角度来看，这种方法之间的引用传递并没有错误，但是执行结果完全取决于方法本身。  
     即当目标方法在接收到一个引用参数时，应该结合业务逻辑，确保不会因此该参数为null值而导致方法执行失败。当然如果业务逻辑允许，推荐将该方法改造为null安全的方法。  
- 对于存在null值的两个引用变量之间，可以使用 == 或者 != 来比较，但是不能使用其他的关系运算符来比较。如下：  
    ```java
    String str = null;
    str == null;// true
    str != null;// false
    str > null;// 编译报错
    str < null;// 编译报错
    ```
     同样，对于存在null值的两个变量之间，其中有一个是基本类型的话，则null值不可以和基本类型发生任何关系运算，否则将抛出空指针异常。比如：  
     ```java
    Integer a = null;
    a == 0;// 空指针
    a != 0;// 空指针
    a > 0;// 空指针
    a < 0;// 空指针
    ....
    ```
     即,null值不能和任何基本类型发生运算。因此往往在比较两个对象时，首先要判断其中是否存在null值的可能。  
- String类的 valueOf() 方法总共重载了9种。valueOf()方法目的是将某种对象转换为String，该方法实际上和String的构造方法类似，都是将某个对象转换为String后返回一个新的String对象。  
    null值如果直接传递给valueOf()方法，比如 String.valueOf(null);则调用的是 public static String valueOf(char data[]) ;因此会抛出空指针异常。  
    Integer i = null;String.valueOf(i);则调用的是 public static String valueOf(Object obj);该方法不会报错，对于null值的引用，会返回字符串“null”。   
- 前面说过，比较两个对象（存在基本类型或者引用类型），如果存在null值的话，则有可能出现空指针异常。替代方案如下：   
    ```java
    // 不论是两个引用类型还是基本类型，该方法都统一装箱为两个object对象，并且内部进行了判空处理，因此可以避免空指针异常。
    Objects.equals(Object o1,Object o2); 
    ```
- 两个null值进行等于比较，比如 null==null将返回true，或者使用Objects.equals()去比较两个null值也返回true。
    
## 1.3 Objects.equals()采坑记录  
Objects.equals()方法，尤其常见于两个基本类型的比较。但是尤其注意：
```java
Long a = 1L;
Integer b = 1;
Objects.equals(a,b); // 将返回false。
```
原理如下：  
Objects.equals()方法源码如下：
```java
public static boolean equals(Object a, Object b) {
    return (a == b) || (a != null && a.equals(b));
}
```
而Long类中的equals()方法如下：
```java
public boolean equals(Object obj) {
    if (obj instanceof Long) {
        return value == ((Long)obj).longValue();
    }
    return false;
}
```
因此，第二个参数为Integer类型，和Long类型不匹配，所以调用Long.equals()方法直接返回false。

# 2. 如何避免NPE？
## 2.1 如何有效避免空指针异常？
- 通过实例对象调用实例方法时，切记要对实例对象进行判空处理。  
- 调用Object类或者任何子类覆盖后的equals()方法比较两个对象是否相等时，比如：word.equals("zhaoeh");  
    极其容易出现NPE，应该改写如下：  
    word != null && word.equals("zhaoeh");  
    "zhaoeh".equals(word);  
    或者直接使用：  
    Objects.equals();  
- 使用StringUtils.isEmpty()或者StringUtils.isBlank()来判断目标字符串是否为null值或者空白串。  
- 判断集合为空，最好使用CollectionUtils.isEmpty()或者MapUtils.isEmpty()方法。  
- 定义方法如果存在返回值，最好不要直接返回null值给调用方。  
- 尤其注意自动拆箱的空指针陷阱。  

# 3. 深入Objects.equals()方法
是的，`Objects.equals` 方法是一个非常安全且通用的工具，可以用来比较任何两个对象，包括以下情况：

1. **两个对象可能为 `null`**  
   如果任意一方为 `null`，`Objects.equals` 不会抛出 `NullPointerException`。

2. **两者均为非 `null`**  
   它会调用第一个对象的 `equals` 方法来比较两个对象。

3. **比较基本类型的包装类**  
   例如 `Integer`、`Double` 等基本类型包装类的比较，可以通过 `Objects.equals` 正确处理。

4. **比较任意类型的对象**  
   如果两者类型不同，它会根据 `equals` 的逻辑返回 `false`。

---

### 实现逻辑解析

`Objects.equals` 的代码实现如下（JDK 源码）：

```java
public static boolean equals(Object a, Object b) {
    return (a == b) || (a != null && a.equals(b));
}
```

**分析**：
- 如果 `a` 和 `b` 是同一个引用（即 `a == b`），返回 `true`。
- 如果 `a` 是 `null`，直接返回 `false`（因为前面已经排除了两者均为 `null` 的情况）。
- 如果 `a` 不为 `null`，调用 `a.equals(b)` 进行比较。

---

### 使用场景

#### 1. **两个可能为 `null` 的对象比较**
手动比较时，通常需要编写额外的代码来检查 `null` 值：

```java
if (a != null ? a.equals(b) : b == null) {
    // 相等
}
```

使用 `Objects.equals` 则简单得多：

```java
if (Objects.equals(a, b)) {
    // 相等
}
```

#### 2. **基本类型包装类的比较**
包装类（如 `Integer`, `Double` 等）中的 `equals` 方法会比较值，但如果其中一个是 `null`，直接调用 `equals` 会抛出 `NullPointerException`。使用 `Objects.equals` 可以避免这个问题：

```java
Integer a = null;
Integer b = 100;

System.out.println(Objects.equals(a, b)); // 输出 false
```

#### 3. **类型不匹配的比较**
当两个对象类型不匹配时，`Objects.equals` 会正确返回 `false`，而不会抛异常：

```java
String a = "test";
Integer b = 100;

System.out.println(Objects.equals(a, b)); // 输出 false
```

#### 4. **数组比较**
需要注意的是，`Objects.equals` 不会比较数组的内容，而是比较数组的引用地址。若要比较数组内容，应使用 `Arrays.equals`：

```java
int[] arr1 = {1, 2, 3};
int[] arr2 = {1, 2, 3};

System.out.println(Objects.equals(arr1, arr2)); // 输出 false（引用不同）
System.out.println(Arrays.equals(arr1, arr2)); // 输出 true（内容相同）
```

---

### 局限性

1. **不适用于深度比较**  
   如果需要比较复杂对象的嵌套内容，应使用工具类（如 Apache Commons Lang 提供的 `EqualsBuilder`）。

2. **不适用于基本类型**  
   基本类型（如 `int`, `double` 等）是无法直接传递给 `Objects.equals` 的，需要使用包装类。对于基本类型，直接使用 `==` 更高效。

```java
int a = 100;
int b = 100;

// Objects.equals(a, b); // 编译错误，不能直接比较基本类型
System.out.println(a == b); // 正确，输出 true
```

---

### 总结

`Objects.equals` 是一个强大的工具，用于安全地比较两个对象。它的特点包括：
- 可以处理任意对象（包括 `null`）。
- 避免了手动处理 `null` 的繁琐逻辑。
- 不抛出 `NullPointerException`。
- 适合大多数场景，但对于数组内容比较和基本类型需要注意。

在日常开发中，推荐优先使用 `Objects.equals` 来简化代码逻辑和提高可读性。
